# Proposal: Skill Auto-Harvest Engine
Generated by: cm-brainstorm-idea
Date: 2026-05-18

## Why we're doing this

### Qualified Problem
**For:** Users who install GreatDeploy
**Who:** Want their AI tools to share skills but currently get zero skills after install
**The:** Skill management system is a **broken promise**
**That:** Automatically discovers and aggregates skills from all installed AI tools
**Unlike:** Current system that requires manual skill hunting per tool
**Our approach:** Background auto-harvest + notification + review UI

### Root Causes:
1. **Tech**: No service scans all tool directories on first run
2. **Product**: No workflow to pull skills from multiple sources into master registry
3. **Design**: No conflict resolution UI when duplicate skills exist

### Impact if NOT addressed:
- App appears non-functional after install
- Users must manually copy skills between tools
- Skills remain siloed, no cross-tool sharing
- Token security risk from careless skill spreading

---

## What's changing

### Options Evaluated
- **Option A (Auto-Harvest)**: Silent discovery, zero friction, hard to audit
- **Option B (Wizard-Driven)**: Guided import, too much friction upfront
- **Option C (Background + Notification)**: Non-blocking discovery, notification, review UI ← **SELECTED**

### Recommendation
Choose Option C for:
1. Respects user attention (non-blocking discovery)
2. Explicit security review before skills spread to other tools
3. Extensible notification infrastructure for future sync updates
